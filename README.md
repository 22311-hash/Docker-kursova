# DriveHub - Премиум Автомобилен Каталог (Docker Compose)

Този проект представлява трислойно уеб приложение за каталог на автомобили и статистика. Проектът е напълно контейнеризиран чрез **Docker** и се управлява лесно с **Docker Compose**.

## 2. Компоненти на Системата

Приложението се състои от три основни контейнеризирани услуги:

1.  **База Данни (`db`)**:
    *   Използва официален образ на **MongoDB (версия 6.0)**.
    *   Съхранява каталога от автомобили персистентно чрез Docker Volume (`mongodb_data`).
    *   Конфигуриран е здравен тест (`healthcheck`), който гарантира, че базата данни е готова да приема връзки, преди бекендът да се стартира.

2.  **Бекенд Услуга (`backend`)**:
    *   Използва лек образ **Node.js (версия 20-alpine)**.
    *   Предоставя REST API за извличане, филтриране, добавяне и изтриване на коли, както и за пресмятане на статистика.
    *   При първоначално стартиране автоматично попълва базата данни с примерни данни за спортни и луксозни автомобили (seeding).

3.  **Фронтенд Услуга (`frontend`)**:
    *   Използва лек уеб сървър **Nginx (на базата на Alpine Linux)**.
    *   Сервира статичните HTML, CSS и JavaScript файлове на потребителя.
    *   Използва съвременен интерфейс с тъмна тема, гласморфизъм ефекти, филтриране в реално време, бърз панел със статистика за автопарка и интерактивна форма за добавяне на нови коли.

## 3. Комуникация между Услугите

Връзката между компонентите е проектирана съгласно добрите практики за контейнеризация:

*   **Контейнерна Мрежа**: Docker Compose автоматично създава виртуална изолирана мрежа, в която контейнерите могат да комуникират помежду си, използвайки имената на услугите си като хостове (DNS resolution).
*   **Бекенд & База Данни**: Бекендът се свързва към базата данни чрез URI адреса: `mongodb://db:27017/drivehub`. Тук хостът `db` съответства директно на името на контейнера за база данни.
*   **Фронтенд & Бекенд (Reverse Proxy)**:
    *   За да се избегнат проблеми с CORS (Cross-Origin Resource Sharing) и да се улесни сигурността, Nginx във фронтенд контейнера е конфигуриран като **Reverse Proxy**.
    *   Всички заявки от браузъра към `/api/*` се прихващат от Nginx и се пренасочват вътрешно в Docker мрежата към `http://backend:5000/api/*`.
    *   По този начин за крайния потребител приложението работи изцяло на стандартния порт `80`, без да е необходимо да се излага порт `5000` на бекенда директно в публичното пространство.

## 4. Инструкции за Стартиране и Управление

Уверете се, че имате инсталиран **Docker Desktop** и че услугата работи.

### Локално Стартиране с Docker Compose

За да сглобите образите и да стартирате проекта локално, изпълнете следната команда в коренната папка на проекта:

```bash
docker compose up --build -d
```

*Флагове:*
*   `--build`: Сглобява наново Dockerfile файловете за фронтенда и бекенда.
*   `-d` (detached mode): Стартира контейнерите във фонов режим.

След успешно стартиране, отворете браузър на адрес: **[http://localhost](http://localhost)**
